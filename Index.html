Core principle

Do not start from code. Start from a verifiable definition that your team and system can execute against.

Why it matters

Transparency, traceability, and less chaos in multi-agent, multi-team development.

What is Docs-as-System

A set of uniform templates, policies, and processes that make docs operational: PROJECT_SPEC.md, PLAN.md, COMPLIANCE.md, TEST_SPEC.md, PR integration, and CI consistency checks.

How it looks in practice

The document is an API of understanding: human and agent readable, with clear sections for I/O, rules, risks, and tests.

Quick start

  • Copy templates/ to your repo.
  • Fill the minimal fields to bootstrap.
  • Run a pipeline that validates code against the specs.

RTM - Right to Maintain

Always-current docs, automated checks, and a merge policy that returns final responsibility to a human.
