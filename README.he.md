# Docs-as-System™

> _Docs-as-System™ היא מתודולוגיית פיתוח חיה, שבה תיעוד, קוד וסוכן המשולב ב־IDE פועלים יחד כמערכת מסונכרנת אחת._

> **תיעוד הוא לא תוצר לוואי – הוא חלק מהמערכת עצמה.**  
> _מתודולוגיית פיתוח מאת [תומר קדם](https://www.linkedin.com/in/tomer-kedem)_  

---

## 🧭 סקירה כללית

**Docs-as-System™** היא מתודולוגיית פיתוח מודרנית שמגדירה מחדש את התיעוד  
כחלק פעיל ומתואם בתהליך הפיתוח עצמו.  
היא מציגה גישה מובנית שבה **תיעוד, קוד וממשל**  
פועלים יחד כישות חיה אחת — קריאה לאדם, נגישה לסוכן, ותמיד מעודכנת.

בלב המתודולוגיה נמצא **הסוכן המשולב ב־IDE** —  
שותף פיתוח חכם שקורא מסמכים, מפרש כוונות, מבצע פעולות,  
ומעדכן את הקוד ואת התיעוד בזמן אמת.  
הסוכן הופך הבנה כתובה למציאות הרצה,  
ומגשר בין החשיבה האנושית לבין התנהגות המערכת.

הרעיון פותח על־ידי **תומר קדם (2025)**  
כמסגרת עבודה מעשית לפיתוח, תיעוד ותחזוקה של מערכות  
מבוססות בינה מלאכותית בעידן הסוכנים החכמים.

> לקובצי המתודולוגיה המלאים ולמשאבים הרשמיים,  
> ניתן לעיין באתר [tomkedem.github.io/Docs-as-System](https://tomkedem.github.io/Docs-as-System)

## 🧩 פילוסופיה

Docs-as-System™ מבססת עיקרון פשוט אך חזק:

> **הבנה באה לפני ביצוע.**  
> כל החלטה, כל סוכן, כל commit —  
> חייבים להתחקות אל כוונה מתועדת.

המתודולוגיה מגדירה שפה חדשה לשיתוף פעולה בין **אדם לסוכן**,  
שבה התיעוד איננו מחשבה מאוחרת אלא ממשק חי למשמעות, שינוי ושליטה.  
כל שינוי במערכת מעדכן את התיעוד,  
וכל עדכון בתיעוד משנה את המערכת — מיידית ובשקיפות מלאה.

## ⚙️ תפקיד הסוכן בסביבת הפיתוח

בפרויקטים מבוססי Docs-as-System™, הסוכן משולב ישירות בתוך סביבת הפיתוח (IDE).  
הוא אינו רק מסייע בקוד — אלא מתאם באופן רציף בין  
**הכוונה (התיעוד)** לבין **הביצוע (הקוד)**.  

המפתח מגדיר את ה"מה" דרך תיעוד חי,  
והסוכן מבצע את ה"איך" ישירות בסביבה.  
כך כל commit הופך לשיקוף מדויק של הבנה מתועדת.
