# מתודולוגיית פיתוח חיה עם סוכן AI משולב ב IDE

> ‏Docs-as-System היא מתודולוגיית פיתוח חיה, שבה התיעוד, הקוד  
> והסוכן הפועל מתוך ה IDE מחוברים יחד כמערכת אחת חיה, מתועדת ומסונכרנת.  
>
> **התיעוד איננו תוצר לוואי אלא חלק מהמערכת עצמה.**  
> ‏מתודולוגיה מאת [תומר קדם](https://www.linkedin.com/in/tomer-kedem)

---

## 🧭 סקירה כללית

‏**Docs-as-System** מגדירה מחדש את מקומו של התיעוד בתהליך הפיתוח.  
היא מציגה גישה שבה **תיעוד, קוד ובקרה** פועלים יחד כישות אחת – כזו שקריאה לאדם, נגישה לסוכן ותמיד מעודכנת.

בלב המתודולוגיה עומד **הסוכן המשולב ב IDE**, שותף פיתוח חכם שקורא מסמכים, מפרש כוונות, מבצע פעולות  
ומסנכרן בזמן אמת בין ההבנה הכתובה לבין המימוש בפועל.

המתודולוגיה משמשת מסגרת עבודה מעשית לפיתוח, תיעוד ותחזוקה של מערכות שמשלבות סוכני בינה מלאכותית בסביבת הפיתוח.

## 🚀 איך מתחילים בפרויקט חדש

כדי ליישם את Docs-as-System בפרויקט חי ולא רק כרעיון תאורטי, קיימים שני רכיבים מעשיים:

• חבילת התחלה דרך npm  
• הרחבה רשמית ל VS Code

### חבילת npm: יצירת פרויקט חדש בפקודה אחת

החבילה הרשמית:

```bash
npx docs-as-system-starterkit-he init my-project
```
# יצירת פרויקט חדש עם **Docs-as-System StarterKit**

הפקודה יוצרת שלד פרויקט מלא שכולל:

• תיקיית **docs** עם תבניות מתודולוגיה מוכנות  
• תיקיית **src** לקוד ההתחלתי  
• קבצי **תצורה חיוניים** של פרויקט  
• קבצי **התחלה לעבודה מסודרת עם הסוכן**  

לפרטים נוספים על החבילה:  
https://www.npmjs.com/package/docs-as-system-starterkit-he

## הרחבה ל VS Code עבודה ישירות מתוך ה IDE

קיימת הרחבה רשמית ל VS Code שמתחברת ל StarterKit  
ומאפשרת ליצור ולהפעיל פרויקטים לפי **Docs-as-System** ישירות מתוך העורך.

קישור להרחבה:  
https://marketplace.visualstudio.com/items?itemName=Tomerkedem.docs-as-system-he

שילוב של ה StarterKit עם ההרחבה יוצר תהליך מלא:

1. יצירת פרויקט חדש בעזרת ה **CLI**  
2. פתיחת הפרויקט ב **VS Code**  
3. עבודה משותפת של מפתח וסוכן על בסיס המסמכים בתיקיית **docs**

## תבניות המתודולוגיה

כל תבניות העבודה הרשמיות של Docs-as-System נמצאות בתיקייה:  
`/methodology-templates`

התיקייה מחולקת לשתי שפות:

• תבניות בעברית: `/methodology-templates/he`  
• תבניות באנגלית: `/methodology-templates/en`  

כל הקבצים כתובים ב **Markdown** ומוכנים לשילוב ישיר ב IDE או בפרויקט קיים.

בפרויקט חדש שנוצר בעזרת ה StarterKit, התבניות הרשמיות מועתקות לתיקיית **docs**  
ומשם משמשות כשפה המשותפת בין האדם לסוכן.

## עקרון הליבה

Docs-as-System מבוססת על רעיון אחד פשוט ומהותי:

**הבנה קודמת לפעולה.**  
כל החלטה, כל סוכן וכל Commit חייבים להתחקות אל **כוונה מתועדת**.

המתודולוגיה יוצרת שפה חדשה לשיתוף פעולה בין אדם לסוכן.  
התיעוד איננו מחשבה מאוחרת אלא **ממשק חי** שמשפיע על הפיתוח בזמן אמת.

---

## תפקיד הסוכן בסביבת הפיתוח

בפרויקטים שפועלים לפי Docs-as-System, הסוכן משולב ישירות ב IDE.  
הוא לא רק מסייע בכתיבת קוד, אלא שומר על **סנכרון רציף** בין הכוונה לבין הביצוע.

המפתח מנסח את ה"מה" במסמך חי, והסוכן מממש את ה"איך" בקוד.  
כך כל Commit הופך לשיקוף מדויק של **הבנה מתועדת**.

בפרויקט שנוצר דרך ה StarterKit, התיקיות והמסמכים מסודרים כך שהסוכן  
יוכל לקרוא אותם, להבין את **הגבולות שלו** ולעבוד לפי המדיניות שהוגדרה.
