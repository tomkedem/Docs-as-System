# A Live Development Methodology with an AI Agent Integrated into the IDE

> **Docs-as-System** is a live development methodology in which  
> documentation, code,  
> and the agent operating inside the IDE are connected as one living, documented, and synchronized system.  
>
> **Documentation is not a byproduct but part of the system itself.**  
> Methodology by [Tomer Kedem](https://www.linkedin.com/in/tomer-kedem)

---

## Overview

**Docs-as-System** redefines the role of documentation in the development process.  
It introduces an approach where **documentation, code, and control** all operate together as one unit  
readable to humans, accessible to the agent, and always up to date.

At the center of the methodology stands the **IDE-integrated agent**,  
a smart development partner that reads documents, interprets intent, performs actions,  
and synchronizes in real time between written understanding and the actual implementation.

The methodology serves as a practical framework for developing, documenting, and maintaining systems  
that integrate AI agents inside the development environment.

## Core Principle

Docs-as-System is built on one simple and essential idea:

**Understanding comes before action.**  
Every decision, every agent, and every commit must trace back to a **documented intent**.

The methodology creates a new collaborative language between human and agent.  
Documentation is not an afterthoughtâ€”it is a **live interface** that shapes development in real time.

---

## The Agent's Role in the Development Environment

In Docs-as-System projects, the agent is integrated directly into the IDE.  
It does more than help write codeâ€”it maintains **continuous alignment** between intent and execution.

The developer writes the *what* in a living document, and the agent implements the *how* in code.  
Every commit becomes an accurate reflection of **documented understanding**.

In a project created using the StarterKit, folders and documents are organized so the agent can read them,  
understand its **boundaries**, and operate according to the defined policy.
