# היסטוריית גרסאות (Changelog)

**מסמך:** `CHANGELOG.md`  
**יוצר:** סוכן פיתוח (לאחר Commit אנושי מאושר)  
**מאשר:** אדם (Tech Lead / מנהל גרסאות / Release Manager)  
**סוג אפיון:** תיעוד שינויים וגרסאות  

---

## הערת פתיח – שימוש אוניברסלי

מסמך זה הוא חלק ממערך התבניות הגנרי של **™Docs-as-System**,  
ומשמש בסיס לתיעוד, בקרה ופיתוח בכל תחום תוכנה –  
Web, Mobile, Backend, Cloud, Data, AI, Embedded או Multi-Agent.  

הוא מניח מודל עבודה שבו הסוכן (AI) הוא המבצע בפועל  
והאדם הוא בעל הסמכות, האישור והבקרה.  
הסוכן פועל תחת מדיניות מתועדת, מתעד כל פעולה,  
ומבצע Commits מקומיים בלבד במערכת ניהול גרסאות (Git או דומה).  
האדם מאשר את השינויים ומבצע את ה-Merge הסופי.  

**עקרון היסוד:**  
הסוכן מיישם → האדם מאשר → Git שומר את האמת.  

---

## מטרת המסמך

לתעד באופן עקבי ומבוקר את כל השינויים שבוצעו בפרויקט לאורך זמן:  
פיתוחים חדשים, תיקוני באגים, שיפורים, עדכוני מסמכים ומדיניות.  

המסמך מהווה **מקור אמת יחיד (Single Source of Change)**  
לכל גרסה של המערכת,  
ומאפשר להבין לא רק מה השתנה – אלא גם למה.

## סוגי שינויים (Change Types)

| סוג שינוי | תיאור |
|-------------|---------|
| **Added** | פיצ'רים או יכולות חדשות |
| **Changed** | שינויים בלוגיקה קיימת |
| **Fixed** | תיקון באגים או שגיאות |
| **Removed** | רכיבים שהוסרו מהמערכת |
| **Documentation** | עדכון מסמכים ותיעוד |
| **Security** | תיקוני אבטחה ו-Hardening |

## דוגמת ערך לגרסה (Version Entry Example)

### 🟢 v1.2.0 – 2025-06-10

**אחראי:** דוד לוי  
**סוכן שותף:** DEV-03  
**סוג:** Minor Release  

#### ✅ Added
- מודול חדש לעיבוד Webhook נכנס.  
- בדיקות Self-Check נוספו למודול Consumers.  

#### 🔄 Changed
- שיפור מנגנון Retry בצד RabbitMQ.  
- עדכון Naming Convention במחלקות Application Layer.  

#### 🐞 Fixed
- תקלה במנגנון Idempotency (Duplicate Key).  
- תיעוד חסר תוקן ב-`PROJECT_SPEC.md`.  

#### 🔐 Security
- הוספת בדיקת Access Token לתקשורת חיצונית.  

#### 📘 Documentation
- עדכון תיעוד של ConsumerAPI והוספת דוגמאות JSON.

## הנחיות כתיבה

- כל גרסה נפתחת בכותרת עם מזהה גרסה ותאריך.  
- סעיפי השינוי מסודרים לפי סדר קבוע:  
  **Added → Changed → Fixed → Removed → Security → Documentation.**  
- תיעוד חייב להיות אנושי, קריא ומובן.  
- הסוכן רשאי לייצר טיוטה אוטומטית לאחר Merge או Release,  
  אך כל עדכון חייב באישור אנושי.  
- אין למחוק גרסאות קודמות – רק להוסיף חדשות.
